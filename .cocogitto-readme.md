# CocoGitto Setup Complete

This project now uses CocoGitto for version management and changelog generation.

## Quick Reference

### Making Commits (use project git aliases - MUCH EASIER!):

```bash
# Features
git feat add new command mapping feature

# Bug fixes
git fix resolve command parsing issue

# Documentation
git docs update installation guide

# Other shortcuts available:
git chore <message>    # Maintenance tasks
git refactor <message> # Code restructuring
git test <message>     # Test updates
git perf <message>     # Performance improvements
git style <message>    # Code formatting
```

**Note:** These aliases are configured **per-project** in `.git/config` and automatically format your commits correctly for CocoGitto!

### Setting Up Aliases in Other Projects:

```bash
# Copy setup-coco-aliases.sh to the project and run:
./setup-coco-aliases.sh

# Or run it from this project:
bash /path/to/claude-hook-advisor/setup-coco-aliases.sh
```

### Version Management:

```bash
# Check if commits are valid
cog check

# See what the next version would be
cog bump --dry-run --auto

# Bump version automatically (runs tests, updates Cargo.toml, creates tag)
cog bump --auto

# Or explicit version bump
cog bump --patch  # 0.2.1 -> 0.2.2
cog bump --minor  # 0.2.1 -> 0.3.0
cog bump --major  # 0.2.1 -> 1.0.0
```

### Changelog:

```bash
# Generate changelog for unreleased commits
cog changelog --unreleased

# Generate full changelog
cog changelog > CHANGELOG.md
```

### Push after bump:

```bash
# CocoGitto creates local tags, you need to push them
git push --follow-tags
```

## Notes

- CocoGitto is a **dev tool** - installed globally, not in Cargo.toml
- Existing commits (before v0.2.1 tag) don't follow conventional format - that's OK!
- All new commits should use `cog commit` to ensure proper format
- The pre-bump hook runs tests automatically before version bumps
